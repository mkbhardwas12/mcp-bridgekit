asyncio.timeout(25s) wraps session.call_tool()

Fast path
Completes in time → SSE stream result to client
⏱
Slow path
Timeout → Enqueue to RQ → Return job_id → Client polls GET /job/{id}
Fast response (< 25s)
Background job (> 25s)
Configurable threshold
Session Pool · Concurrency Model
🔒 Global Lock
A single asyncio.Lock protects lock-map creation. Held only for nanoseconds — just to look up or create per-user locks. Prevents two concurrent first-requests for the same user_id from creating separate locks.
async with self._global_lock:
  if user_id not in self.locks:
    self.locks[user_id] = asyncio.Lock()
  return self.locks[user_id]
🔑 Per-User Locks
Each user_id gets its own lock. All session operations (create, reuse, expire, close) are serialized per user. Different users run fully concurrently.
Lock: user-abc
→ Session 1
Lock: user-xyz
→ Session 2
Lock: user-def
→ Session 3
Pool limits: max_sessions=100 · TTL: 3600s · Eviction: oldest first

Configuration · Environment Variables
Variable (prefix: MCP_BRIDGEKIT_) Default Description
REDIS_URL redis://localhost:6379 Redis connection for jobs + status
MAX_SESSIONS 100 Maximum concurrent MCP sessions in pool
SESSION_TTL_SECONDS 3600 Session expiry — 1 hour default
TIMEOUT_THRESHOLD_SECONDS 25.0 Seconds before a tool call is queued as background job
JOB_RESULT_TTL_SECONDS 600 How long job results stay in Redis (10 min)
DEFAULT_MCP_COMMAND python Default command to run MCP server
DEFAULT_MCP_ARGS ["examples/mcp_server.py"] Default args for MCP server command
